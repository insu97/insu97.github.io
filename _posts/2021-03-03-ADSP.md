## 데이터

1 . 데이터의 특성

	- 존재적 특성 : 객관적 사실
	- 당위적 특성 : 추론 * 예측 * 전망 * 추정을 위한 근거 ( = 주관적 )

2 . 데이터의 유형

	- 정성적 데이터 : 많은 비용이 소모 ( ex . 언어 , 문자 )
	=> 처리하기 어려운 데이터.<br>
	- 정량적 데이터 : 비용 소모가 적음 ( ex . 수치 , 도형 , 기호 )
	=> 비교적 처리하기 쉬운 데이터<br>

3 . 지식경영

	- 암묵지 : 개인의 지식
	- 형식지 : 집단의 지식

4 .  개인정보 보호

	- 데이터 마스킹
	- 가명처리
	- 총계처리
	- 데이터값 삭제
	- 데이터 범주화

5 . 데이터의 유형
```
 - 정형데이터 : 관계형 데이터 베이스에 저장
 - 반정형데이터 : 파일로 저장
 - 비정형데이터 : NOSQL에 저장
```

6 . DIKW
```
  - Data : 개별 데이터 자체로는 의미가 중요하지 않음.
  - Information : 데이터의 가공 , 처리와 데이터간 연관관계 속에서 의미가 도출된 것.
  - Knowledge : 결론과 개인적인 경험을 결합시켜 고유의 지식으로 만듬.
  - Wisdom : 지식의 축적과 아이디어가 결합된 창의적인 산물.
```


## 데이터베이스

1 . 데이터베이스의 특징

	- 통합된 데이터 : 데이터 중복을 허용하지 않음.
	- 저장된 데이터 : 데이터를 저장.
	- 공용 데이터 : 여러 사용자가 데이터 사용 가능.
	- 변화되는 데이터 : 데이터 삭제 , 갱신 가능.

2 . 용어

	- OLTP(On-Line Transaction Processing) : 호스트 컴퓨터가 DB를 엑세스하고 , 바로 처리 결과를 돌려보내는 형태.
	- OLAP(On-Line Analytical Processing) : 다차원적인 데이터에 접근하여 의사 결정에 활용할 수 있는 정보를 얻을 수 있게 해주는 기술.
	- CRM(Customer Relationship Management) : '고객관계관리'
	- SCM(Supply Chain Management) : '공급망 관리'



## 빅데이터

1 . 빅데이터의 이해

	- 3V :  양(Voume) , 다양성(Variety) , 속도(Velocity) + 4v(가치(Value) , 시각화(Visualization) , 정확성(Veracity))

2 . 빅데이터의 위험 요소

	- 사생활 침해 , 책임 원칙 훼손 , 데이터 오용

3 . 집중해야 하는 점

	- 단순히 빅데이터에 집중하지 말고 분석을 통해 가치를 만드는 것에 집중해야 한다.
	- 데이터는 크기의 이슈가 아니라 , 어떤 시각과 통찰을 얻을 수 있는가의 문제.
	- 일차적인 분석은 새로운 기회를 포착하기 어렵다.

## 분석기획

1 . 정의

	- 실제 분석을 수행하기에 앞서 분석을 수행할 과제를 정의
	- 의도했던 결과를 도출할 수 있도록 이를 적절하게 관리할 수 있는 방안을 사전에 계획하는 일련의 작업.

2 . 목표 시점 별 분석 기획 방안

	- '과제 중심적인 접근 방식' : 당면한 과제를 빠르게 해결해야 할 때
	- '장기적인 마스터 플랜 방식' : 지속적인 분석 내 재화를 위한 방식.

3 . 분석 기획시 고려사항

	- 가용 데이터(Available data) : 데이터 확보가 우선적
	- 적절한 활용방안과 Use Case탐색 필요
	- 장애요소들에 대한 사전계획 수립 필요 ( Low Barrier Of Execution )

## 분석 방법론

1 . 모델

```
 - 폭포수 모델 : 단계를 순차적으로 진행하는 방법.
 - 프로토타입 모델 : 고객의 요구를 들어주기 위한 모델 , 일부분을 우선 개발 -> 개선 작업.
 - 나선형 모델 : 반복을 통해 점증적으로 개발하는 방법 -> 단점은 복잡도가 상승할 수 있다.
 - KDD(Knowledge Discovery in Database) : 통계적 패턴이나 지식을 찾기 위한 모델.
   	- 데이터 선택 : 목표를 위해 데이터 선정을 잘 해야 한다.
      	- 데이터 전처리 : 잡음 , 이상치 , 결측치를 처리
      	- 데이터 변환 : 차원 축소 , train_data , test_data 분리
      	- 데이터 마이닝
      	- 데이터마이닝 결과 평가

 - CRISP-DM(Cross Industry Standard Process for Data Mining)분석 방법론
    : 계층적 프로세스 모델
    - 업무이해 : 목적과 요구사항을 이해하기 위한 단계
    - 데이터 이해 : 데이터 수집하고 속성을 이해하는 단계
    - 데이터 준비 : 데이터 셋 선택 , 데이터 정제
    - 모델링 : 모델링 기법과 알고리즘을 선택 -> 최적화 , 과적합 문제 확인
    - 평가 : 프로젝트 목적에 부합하는지 평가
    - 전개 : 실제 업무에 적용하기 위해 계획을 수립하는 단계 , 유지보수 계획도 마련.

 - 빅데이터 분석 방법론
    - 단계(Phase) , Task , Step
    - 분석기획 -> 데이터 준비 -> 데이터 분석 -> 시스템 구현 -> 평가 및 전개

```

## 분석 과제

1 . 하향식 접근법

	- 문제를 탐색 -> 문제 정의 -> 해결방안 탐색 -> 타당성 평가 -> 분석 과제를 도출

2 . 상향식 접근법

	- 다양한 원천 데이터를 대상으로 분석을 수행하여 가치있는 모든 문제를 도출하는 일련의 과정
	- 비지도 학습 , 지도 학습
	- 프로토타이핑 접근법 : 일단 분석을 시도 -> 결과를 확인하여 반복적으로 개선해 나가는 방법.
	 --> 문제를 이해하고 , 이를 바탕으로 구체화 하는데 도움을 받을 수 있다.
